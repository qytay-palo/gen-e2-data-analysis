## Contributing

When creating or updating user stories:

1. **Follow INVEST Principles**: Independent, Negotiable, Valuable, Estimable, Small, Testable
2. **Use Vertical Slicing**: Each story delivers end-to-end analytical value
3. **Reference Domain Knowledge**: Link to domain knowledge files for context and feature guidance
4. **Document Thoroughly**: Include acceptance criteria, constraints, tasks, and notes
5. **Maintain Consistency**: Follow established format and structure
